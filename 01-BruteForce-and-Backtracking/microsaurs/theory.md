# 브루트포스 알고리즘

가능한 모든 경우의 수를 탐색하면서 요구조건에 충족되는 결과를 가져오는 알고리즘  
무식한 힘, 노가다 .. 등으로 이해하면 됨

**장점**  
모든 경우의 수를 전부 탐색하여 조건에 맞는 답을 찾기 때문에 정환학 답을 보장  
알고리즘을 설계하고 구현하기 매우 쉬움  
복잡한 알고리즘 없이 빠르게 구현

**단점**  
시간 복잡도가 높음  
메모리 효율이 매우 비효율적

✔️ 브루트포스 알고리즘은 정형화된 틀이 없기 때문에 **문제 유형에 따라 다양한 방법으로 풀이**

- 선형 구조 : 순차 탐색
- 비선형 구조 : 백트래킹, DFS, BFS

### 브르투포스 알고리즘 사용하는 이유

입력되는 범위가 작고, 탐색해야할 경우의 수가 적은 간단한 문제의 경우 최적화된 복잡한 알고리즘을 구현하는 비용이 높기 때문에 직관적으로 모든 경우의 수를 구하는 쪽이 더 효율적  
모든 경우를 탐색하여 답을 도출하므로 항상 정확한 답을 찾아낼 수 있는 강력한 방법
